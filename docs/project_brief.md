# Project Brief: RISC-V CPU Verification

## 1. Project Question

How can we verify that a simplified RISC-V CPU executes instructions correctly?

## 2. Background

Modern chips are extremely complex. Before a processor is manufactured, engineers must verify that it behaves correctly under many different situations.

This project introduces students to the basic idea of CPU verification using a simplified C++ model.

## 3. Student Role

Students act as junior verification engineers. Their job is to:

- understand basic CPU behavior
- run test programs
- compare actual results with expected results
- identify bugs or incorrect behavior
- document the verification process

## 4. Key Engineering Ideas

- A CPU executes instructions step by step
- Registers store temporary values
- Memory stores data
- A testbench checks whether output is correct
- Debugging is a key part of engineering

## 5. Final Deliverables

- C++ CPU testbench
- assembly-style test programs
- test result summary
- engineering report
- final presentation
